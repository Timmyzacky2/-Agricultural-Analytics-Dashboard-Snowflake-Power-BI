# Agricultural-Analytics-Dashboard-Snowflake-Power-BI

### Dashboard Project

An end-to-end agricultural analytics solution built using **Snowflake** and **Power BI** to analyze environmental factors affecting crop productivity. The project demonstrates a modern cloud analytics workflow where raw agricultural data is stored and transformed in Snowflake before being visualized through interactive Power BI dashboards.

---

# 📌 Project Overview

Agriculture is heavily influenced by environmental conditions such as rainfall, temperature, and humidity. Understanding how these factors impact crop yields enables farmers, agricultural planners, and policymakers to make informed decisions that improve productivity and resource allocation.

This dashboard provides comprehensive insights into agricultural performance by analyzing environmental trends and crop yield patterns across multiple years, seasons, locations, and crop categories.

The solution combines Snowflake's scalable cloud data warehousing capabilities with Power BI's powerful visualization features to deliver actionable insights.

---

# 🎯 Business Problem

Agricultural organizations often collect large amounts of environmental and crop production data but struggle to transform that information into meaningful insights.

Without proper analysis, it becomes difficult to:

* Identify environmental conditions affecting crop production
* Monitor long-term climate trends
* Compare agricultural performance across regions
* Optimize farming strategies
* Improve resource allocation and planning

This dashboard addresses these challenges by providing a centralized analytical view of environmental and crop performance metrics.

# ⚙️ Data Preparation Process

## Step 1: Data Ingestion

The agricultural dataset was imported into Snowflake for centralized storage and processing.

## Step 2: Data Cleaning in Snowflake

Data quality issues were addressed using Snowflake SQL transformations.

Cleaning activities included:

* Removing duplicate records
* Handling missing values
* Standardizing data formats
* Validating numerical fields
* Correcting inconsistent values
* Preparing analysis-ready datasets

## Step 3: Data Validation

The transformed dataset was validated to ensure:

* Data accuracy
* Data completeness
* Consistent aggregation results
* Reliable reporting outputs

## Step 4: Power BI Integration

Power BI was connected directly to Snowflake to retrieve the cleaned and transformed data.

## Step 5: Data Modeling

Relationships and analytical structures were optimized to improve dashboard performance and reporting efficiency.

## Step 6: Dashboard Development

Interactive visualizations were created to analyze:

* Rainfall patterns
* Temperature trends
* Humidity levels
* Crop yield performance
